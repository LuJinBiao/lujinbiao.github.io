6-面向对象(下)

发表于 2018-03-27 | 更新于 2018-06-17 | 分类于 Java

包装类

Java 提供了 8 种基本数据类型的对应包装类，并提供了 自动装箱 和 自动拆箱 功能。

自动装箱 就是可以把一个基本类型变量直接赋给对应的包装类变量，或者赋给 Object 变量。

自动拆箱 允许直接把包装类对象直接赋给一个对应的基本类型变量。

系统把一个 -128 ~ 127 之间的整数自动装箱成 Integer 实例，并放入了一个名为 cache 的数组中缓存起来。如果以后把一个 -128 ~ 127 之间的整数自动装箱成一个 Integer 实例时，实际上是直接指向对应的数组元素，因此在该范围的同一个整数自动装箱成 Integer 实例时，永远都是引用 cache 数组的同一个数组元素，因此 == 运算永远相等。

包装类实现了基本类型变量和字符串之间的转换，把字符串类型的值转换为基本类型：

  • 利用包装类提供的 parseXxx(String s)。
  • 利用包装类提供的 Xxx(String s) 构造器。

String 类提供了多个重载 valueOf() 方法，用于将基本类型变量转换成字符串。

理解类成员

Java 类里面只能包含 成员变量、方法、构造器、初始化块、内部类(包括接口、枚举) 5 种成员。

static 可以修饰 成员变量、方法、初始化块、内部类(接口、枚举)，以 static 修饰的成员就是类成员，属于整个类，而不属于单个对象。

对 static 关键字而言，有一条非常重要的规则：类成员(包括 方法、初始化块、内部类和枚举类)不能访问实例成员(包括成员变量、方法、初始化块、内部类和枚举类)。

单例类

如果一个类始终只能创建一个实例，则为单例类。为了避免其他类自由创建该类的实例，应该把该类的构造器使用 private 修饰，从而把该类的所有构造器隐藏起来。

一旦把该类的构造器隐藏起来，就需要提供一个 public 方法作为该类的访问点，用于创建该类的对象，且该方法必须使用 static 修饰。除此之外，该类还必须缓存已经创建的对象，否则该类无法知道是否已经创建过对象。为此该类需要使用一个静态成员变量来保存曾经曾经的对象。

关于 final

final 修饰基本类型变量时，一经赋值就不能再改变。
final 修饰引用类型变量时，只保证这个引用类型变量所引用的地址不会改变，即一直引用同一个对象，但这个对象的内容可以发生改变。

final 修饰方法使其不可被重写，多用于不希望子类重写父类的某个方法。同理，final 修饰类使其不可被继承。

不可变类 指创建该类的实例后，该实例的实例变量是不可改变的。要创建自定义的不可变类，有规则如下：

  • 使用 private 和 final 修饰该类的成员变量。
  • 提供带参数的构造器，用于根据传入参数来初始化类的成员变量。
  • 仅为该类的成员变量提供 getter 方法，不要为该类的成员变量提供 setter 方法。
  • 如有必要，重写 hashCode() 和 equals() 方法。

抽象类和接口

抽象类

  • 抽象类必须使用 abstract 修饰，抽象类可以没有抽象方法，但有抽象方法的类必须是抽象类。
  • 抽象类不能被实例化，即使抽象类里没有抽象方法。
  • 抽象类里可以包含成员变量、方法(普通方法和抽象方法都可以)、构造器、初始化块、内部类(接口、枚举)5种成分。抽象类的构造器不能用于创建实例，主要是用于被其子类调用。

final 和 abstract 永远不能同时使用；static 和 abstract 不能同时修饰某个方法。

总结：抽象类是从多个具体类中抽象出来的父类，它具有更高层次的抽象。从多个具有相同特征的类中抽象出一个抽象类，以这个抽象类作为其子类的模板，避免了子类设计的随意性。

抽象类体现的就是一种模板模式的设计，抽象类作为多个子类的通用模板，子类在抽象类的基础上进行扩展、改造，但子类总体上会大致保留抽象类的行为方式。

接口

接口 是一种功能扩展。接口不能包含普通方法，接口里的所有方法都是抽象方法。Java8中允许在接口定义默认方法(使用 default 修饰)，默认方法可以提供方法实现。

接口定义的是多个类共同的公共行为规范，接口里通常是一组公用方法。一个接口可以有多个直接父接口，但只能继承接口，不能继承类。

接口定义的是一种规范，因此接口里不能包含构造器和初始化块定义。接口里可以包含成员变量(只能是静态常量)、方法(只能是抽象实例方法、类方法或默认方法)、内部类(包括内部接口、枚举)。

接口允许支持多继承，即一个接口可以有多个直接父接口。

一个类实现一个或多个接口之后，这个类必须完全实现这些接口里所定义的全部抽象方法。并且该类将会获得接口中定义的常量、方法等。
