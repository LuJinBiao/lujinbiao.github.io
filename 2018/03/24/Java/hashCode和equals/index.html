正确地重写 equals() 方法应该满足下列条件：
自反性：对任意 x，x.equals(x) 一定返回 true。
对称性：对任意 x 和 y，如果 y.equals(x) 返回 true，则 x.equals(y) 也返回 true。
传递性：对任意 x，y，z，如果 x.equals(y) 返回 true，y.equals(z) 返回 true，则 x.equasl(z) 一定返回 true。
一致性：对任意 x 和 y，如果对象中用于等价比较的信息没有改变，那么无论调用 x.equals(y) 多少次，返回的结果应该保持一致，要么一直是 true，要么一直是 false。
对任何不是 null 的 x，x.equals(null) 一定返回 false。

对于数组类型的域，可以使用静态的 Arrays.equals 方法检测相应的数组元素是否相等。

重写 hashCode()

重写 hashCode() 方法的基本规则：

  • 同一个对象多次调用 hashCode() 方法应该返回相同的值。
  • 当两个对象通过 equals() 方法比较返回 true 时，这两个对象的 hashCode() 方法应该返回相等的值。
  • 对象中用作 equals() 方法比较的实例变量，都应该用于计算 hashCode 值。
