SpringMVC-信息转换

发表于 2018-05-23 | 更新于 2018-11-10 | 分类于 Spring

HttpMessageConverter 接口

HttpMessageConverter 接口负责将请求信息转换为一个对象(类型为 T)，并将对象绑定到请求方法的参数中或输出为响应信息。

DispatcherServlet 默认已经装配了 RequestMappingHandlerAdapter 类作为 HandlerAdapter 组件的实现类，在 RequestMappingHandlerAdapter 类中使用了 HttpMessageConverter，将请求信息转换为对象。

Spring 为 HttpMessageConverter 提供了多个实现类，以下是部分实现类：

  • StringHttpMessageConverter：将请求信息转换为字符串。
  • FormHttpMessageConverter：将表单数据读取到 MultiValueMap 中。
  • XmlAwareFormHttpMessageConverter：继承自 FormHttpMessageConverter ，如果部分表单属性是 XML 数据，则可用该转换器进行转换。
  • ResourceHttpMessageConverter：读写 org.springframework.core.io.Resource 对象。

Converter 和 Formatter

Converter 和 Formatter 的作用是将一种类型转换成另一种类型的一个对象。例如，如果希望 Spring 在将输入的日期字符串绑定到 Date 时，使用不同的样式，则需要编写一个 Converter，才能将字符串转换成日期。

创建 Formatter

Formatter 的作用跟 Converter 一样也是进行类型转换，但是 Formatter 的源类型必须是一个 String，而 Converter 则适用于任意的源类型。因此 Formatter 更适合于 Web 层，而 Converter 更适合用在任意层中。

更多时候，Formatter 接口用来把客户端传来的 String 字符串转换成具有一定格式的数据，比如 日期、时间、数字、货币，在不同的本地环境中，同一类型的数据还会相应的呈现不同的显示格式。

区别与联系

ConversionService 是属于spring-core 部分所有 spring 管理的 bean 等等都使用它进行类型转换，而它又需要注册一些 converter 来完成类型转换。

HttpMessageConverter 是对 http请求/响应 数据进行转换的， 它可以使用 ConversionService 进行一些转换（HttpMessageConverter是否能转换 还需要根据如请求的contentType等决定）.
